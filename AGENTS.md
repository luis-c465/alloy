# Alloy - Agent Reference

## What is Alloy?

Alloy is a Rust-powered desktop HTTP API client built with Tauri v2, React 19, and TypeScript. Think Postman/Insomnia but native, fast, and file-based. Users compose HTTP requests, send them through a Rust backend (Reqwest), and inspect responses.

## Architecture

```
React 19 UI --> Zustand Store --> TauRPC Client
    --> [IPC Bridge] -->
TauRPC Rust Handler --> Reqwest --> Target HTTP Server
    --> [Response back through same chain] -->
Response Viewer UI
```

**Frontend:** React 19 + TypeScript, Zustand state, CodeMirror 6 editors, shadcn/Radix UI, Tailwind CSS 4, react-resizable-panels for IDE-like layout.

**Backend:** Tauri v2 + Rust. TauRPC generates TypeScript bindings from Rust traits (type-safe IPC). Reqwest for HTTP. rusqlite for history. Handlebars for `{{variable}}` templating.

**Layout:** Three-panel IDE layout — collapsible sidebar (collections/history), tab bar + request builder (top), response viewer (bottom).

## Key Patterns

### TauRPC IPC

Each domain has a Rust trait in `src-tauri/src/commands/`:

```rust
#[taurpc::procedures(path = "workspace", export_to = "../src/bindings.ts")]
pub trait WorkspaceApi {
    async fn list_files(workspace_path: String) -> Result<Vec<FileEntry>, AppError>;
    // ...
}
```

Frontend calls via `src/lib/api.ts`:

```typescript
import { createTauRPCProxy } from "~/bindings";
const api = await createTauRPCProxy();
const files = await api.workspace.list_files(path);
```

All TauRPC traits use `export_to = "../src/bindings.ts"` -- types merge into one generated file. **Never hand-edit `src/bindings.ts`.**

### Zustand Multi-Tab Store

The main store (`request-store.ts`) uses a `tabs: Tab[]` array with `activeTabId`. Each `Tab` holds its own method, URL, headers, body, response, loading state, etc. Setter actions (e.g. `setMethod`) operate on the active tab.

Convenience: `useActiveTab()` hook returns the active tab object.

Race condition note: `sendRequest` captures the tab ID at invocation time so responses go to the correct tab even if the user switches tabs mid-flight.

### .http File Format

Requests are stored as `.http` files (VSCode REST Client compatible):

```http
@base_url = https://api.example.com

### Get Users
# @name GetUsers
GET {{base_url}}/users HTTP/1.1
Authorization: Bearer {{token}}

### Create User
# @name CreateUser
POST {{base_url}}/users HTTP/1.1
Content-Type: application/json

{"name": "Alice"}
```

### Workspace Layout

```
my-project/
  .alloy/
    config.toml              # active_environment = "local"
    environments/
      local.toml             # [variables] base_url = "http://localhost:3000"
      production.toml
  requests/
    users.http
    auth.http
```

### Environment Variables

TOML files in `.alloy/environments/`. Variables use `{{name}}` syntax (Handlebars). Resolved at send-time only -- the editor always shows raw templates. Resolution applies to URL, headers, query params, and body content.

## Build & Dev Commands

```bash
bun install                  # Install frontend deps
bun run dev                  # Vite dev server only (no Tauri)
bun tauri dev                # Full app: Rust + frontend (generates bindings.ts)
bun tauri build              # Production build
cargo build                  # Rust only (from src-tauri/)
cargo test                   # Rust tests (from src-tauri/)
cargo check                  # Fast type check (from src-tauri/)
```

**Important:** `bindings.ts` is generated at Rust **runtime** (when the binary executes), not at `cargo build` time. Run `bun tauri dev` at least once to generate it.

bunx shadcn@latest add <component>

## Critical Gotchas

1. **specta version pin:** `specta = "=2.0.0-rc.22"` -- must be exact, not caret range.

2. **`#[taurpc::ipc_type]`** auto-derives Serialize, Deserialize, specta::Type, Clone. Do NOT manually derive these or you get duplicate trait errors.

3. **AppError serialization:** Must implement `serde::Serialize` manually (not derive) because inner types like `reqwest::Error` don't implement Serialize.

4. **TauRPC + AppHandle:** TauRPC resolvers can't inject `AppHandle` as a parameter directly (it's treated as a deserializable arg). Shared state is passed via `Arc<OnceCell<...>>` on resolver structs.

5. **RequestBody enum in TypeScript:** TauRPC/specta serializes Rust enums as tagged unions. Check `bindings.ts` for the exact shape (e.g., `{ type: "Json", data: "..." }`).

6. **BigInt export:** Specta fails on `u64` by default. The router export config uses `BigIntExportBehavior::Number` (or `String`).

7. **`bindings.ts` strict mode:** Generated file includes `// @ts-nocheck` header because `noUnusedLocals` in tsconfig would otherwise fail on internal helper types.

8. **Path alias:** TypeScript uses `~/` as path alias for `./src/` (configured in tsconfig).

9. **react-resizable-panels v4:** Exports are `Group`, `Panel`, `Separator` (not `PanelGroup`/`PanelResizeHandle`).

10. **History DB location:** Platform-specific app data dir (`~/.local/share/com.luisc.alloy/` on Linux).

## Current State

All four phases are implemented:
- **Phase 1 (MVP):** Core request/response loop
- **Phase 2:** Workspace, collections (.http files), environments (TOML), history (SQLite), multi-tab UI
- **Phase 3:** Multipart uploads, auth presets (Bearer/Basic), cookie viewer, SSL/TLS options, per-request timeout
- **Phase 4:** Response search/filter, binary handling, cURL import/export, Postman import, keyboard shortcuts, light/dark theme
